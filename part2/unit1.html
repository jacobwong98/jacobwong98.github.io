How the Web Works

The World Wide Web (WWW) allows computer users to execute web-based app and locate/view multimedia-based documents over the Internet. Here are some key technologies that explain how the users are able to interact with the World Wide Web:

  • : Communication protocols that send information over the web
  • : URLs of websites that handle private information
  • : Specifies address/location of web page that is displayed in browser window

Another important definition is the Uniform Resource Identifier (URI).
URIs help identify resources on the Internet and URIs that start with http:// are URLs

Making a Request and Receiving a Response

In order to allow users to interact with websites, we need to understand how pages are displayed to users and information is passed between users & websites

HTTP Requests

  • : Retrieves information from a web server. E.g. User enters a URL in the web browser which sends a GET request to the server to display whatever is at the URL location.
  • : Send data to a web server. E.g. User fills a sign-up form online and the information is sent to the server for processing.
Once these requests are sent to the server, the server will perform the request and send some information back to the web client (normally the user's browser).
Included in this information is a numeric code that represents the "outcome" of the request.
If the code is 200, then the server was successful in returning the requested resource to the browser. If the code is 404, this means the server was unable to locate the resource that was requested.

Now you may be wondering what these "resources" are that the server can display for the user and these include images, videos and most importantly HTML documents

HTML

What is HTML? HTML or Hypertext Markup Language is a markup language that desribes the structure and content of documents displayed in web browsers. The difference with HTML and most other languages like C, C++, Python, Java is that HTML is strictly deals with structure/content only and in combination with CSS (to be shown soon) and JavaScript (See Unit 2) form the basics of websites on the Internet.

Basic Components of HTML

Throughout this tutorial I will use the terms "tags" and "elements".
Tags are notation that determine how content within a start/end tag should behave (or sometimes no end tag).
Elements go within the tags and is the behaviour of the content

CSS

Cascading Style Sheets (CSS) is used for presentation of elements, separately from the structure of the document. In order to customize the presentation of documents, we can add CSS selectors & rules.

For example, h1, .myClass, #uniqueID { property: value; }
  • represents what styling you want to change (e.g. color, border, margin, etc)
  • represents the value that the property will become (e.g. red, 10px, 75%, etc)
  • means we want to apply the CSS to h1 element. You can replace h1 with any element
  • indicates a class attribute is selected (indicated by the . before the name of the class)
  • indicates the id attribute is selected (indicated by the # before the name of the id

You can have as many pairs of properties and values within the curly braces; these represent the rules that apply to the selectors before the curly braces.
Now that we know how CSS works, we can combine CSS and HTML in 3 ways:

  • : Within HTML add the style attribute. E.g. <span style="font-family: TimesNewRoman">Times New Roman</span>
  • : Add the CSS to an HTML document within style element. E.g. <style> span { font-family: 'Times New Roman'; } </style>
  • : Add CSS like Embedded Style Sheets (except no style tags) in a separate CSS file. Then add <link href="styles.css" type="text/css" rel="stylesheet" /> to the HTML document (most likely in the head element)
