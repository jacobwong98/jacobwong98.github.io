In Unit One, we looked at how we can use HTML to determine the structure/content of webpages using markup language (i.e. elements provided like p, span, b, etc).
With Extensible Markup Language (XML) , it allows us to create our own markup languages to represent any kind of information. For example

<book> <title>Harry Potter and the Deathly Hallows</title> <author>J.K Rowling</author> </book>

XML can also allow attributes just like HTML inside the tags that help add information about the tags. In the above example I could add the attribute bookNum="7" within the title tag and this would show that the Deathly Hallows was the 7th book of the series.

Some basic rules/practices for XML

  • Should use the XML declaration to identify an XML document and its version of syntax (optional)
  • Elements must had start and end tags & element names must start with letter/underscore and xml
  • Adding whitespace in the element name is an error
  • Necessary to have a root element that encompasses all other elements
  • helps prevent issues differentiating elements when they have the same element name. For example, <hockey:player>Wayne Gretsky</hockey:player> <basketball:player>Michael Jordan</basketball:player> allows the XML parser to know that the first element is a player in hockey while the later is a player in basketball

DTDs and Schemas

Document Type Definitions (DTDs) and schemas specify the element type/attributes and relationships to one another in an XML document.
This helps the XML parser to verify that a XML document is valid (could be structured properly but is not valid).
DTD expresses rules for document structure using Extended Backus-Naur Form (EBNF) grammar. The issue with DTD is that it can not describe the element or attribute's data type.

XML Schemas don't use EBNF grammar however they are able to specify the type of data an element can contain. Much like DTDs, XML documents that conforms the schema is considered to be valid.
With schemas, we can have two types of elements: Simple & Complex where one can't contain child elements and defines restrictions on schema-type or user-defined type. Whereas, complex types can have attributes/child elements where the child elements can be simple or complext types.

So now our XML is valid but when you open your XML in a browser it still displays the same structure that is shown in a text editor. Let's find out how to make your XML styled how you want on the browser.

Extensible Stylesheet Language

Extensible Stylesheet Language (XSL) can be used to convert XML into a text-based document (i.e. into content that can be displayed in the browser). Much like CSS, we can apply an XSL rules save in a .xsl file to an XML document like this:

<?xml -stylesheet type="text/xsl" href="resume.xsl" ?>

Here is what goes in the file to do what we want:

  • / character - Selects the document root (one of the reasons that we need a root element in XML)
  • XSL element value-of - Gets an attribute's value
  • match="" - Selects depending on what is in the double quotes (can be elementName, a path to an element, etc)

Ajax

Asynchronous JavaScript and XML (Ajax) allows web applications to be more responsive using client-side scripting (JavaScript). How does it make them more responsive? Ajax separates client-side interaction and server communication by running them in parallel. This means when a request is sent to the server, the user can still interact with page without having the application to wait for the server to respond to the request.

What Ajax does is use the XMLHTTPRequest object to handle the request/response from the server. Then when the server sends the response back, the XMLHTTPRequest object invokes a callback function that will update a portion of the page so that the entire page does not need to be reloaded. This callback function is registered as an event handler (readystatechange) for the XHR object and is called whenever the request makes progress. Here are the steps for making an XMLHTTPRequest

  1. Create an XMLHTTPRequest object
  2. Add readystatechange event listener to XHR object
  3. Use open method to initilize request with method (GET or POST) and URI
  4. Use send method to send the request to the server
  5. Within the function that will run on readystatechange, check for:
    • readyState == 4 (Request is complete)
    • status == 200 (Server successfully returned response)
  6. Can then use responseText or responseXML to get the content requested depending on what you are requesting
