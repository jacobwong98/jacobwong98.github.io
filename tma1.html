Original Description and Requirements

  • Part 1: Create an XML version of your resume containing at least three sections: general information about you; your educational background, and your work experience. Create a schema for the XML document, and then create an XSLT for the XML document so that it can be rendered in a browser.

    2. The application must have three pages of tutorials, accessible through navigation menus or buttons, to teach the technologies covered in Unit 1, Unit 2 and Unit 3 respectively. You don’t need to teach everything, but there should be enough content to make each of the tutorials useful.

    6. You should do your best to use the web technologies covered in Unit 1, unit 2 and Unit 3, including HTML5, CSS3, JavaScript, XML and Ajax.
    7. Your web application should use or teach at least five page-structure elements and six new input types

    9. When using HTML5 table, be sure that you only use it to display tabular information, not to lay out the elements on your page.
  • Part 3: For this project, you are required to create a slideshow web application using HTML5 canvas and other web technologies you have learned so far. The requirements are as follows:

    3. There should be a button to start/stop the show
    4. There should be a control to toggle the show between random and sequential
    5. There should be buttons to manually turn the show backward or forward, only if the show is in sequential mode
    6. There should be a dropdown list for users to select different transition/transformation effects for the show
    7. The data of the images should be in a JSON array so that images and their captions can be easily maintained
    8. To avoid copyright issues, you should use the images/photos you took. If you don’t have the images ready, please go around the place you live or work and take some good ones
    9. The show should have at least 20 nice pictures.

    • Measurement converters: weights, lengths, areas, and volumes
    • A mortgage calculator
    • A utility tool that you think could be useful. You may need to do some research on the Web to build this last tool, and in the report you need to justify why you think it will be useful.

    1. It must have an integrated and user-friendly interface for users to access each of the tools you designed.
    2. You need to use JavaScript functions and built-in objects to implement the application.
    3. At any time only one tool will be shown, to ensure a clean and tidy space for users to work with.

    6. In the assignment report, you need to provide sufficient documentation for your work, including algorithms and formulas used for the tools. If you have used any resources, including documents on the Web, you must clearly identify the source in your documentation.

Interpretation & Implementation

  1. Part 1: The requirements were pretty straight-forward so I simply looked at my resume and converted what I could to XML format first then completed the XSLT by trying to format the same way that my resume looked.
  2. Part 2: I started by using the notes I had made about all the Units so far and took the content I needed to start making my tutorial in HTML and CSS. Then when I finished putting the content in for Unit 1 I made my quiz JavaScript that would load/mark the questions using Ajax to get the XML that stored my quiz. I made my script generic so that any quiz could use the script without having to change it. Because the description said to use HTML, CSS, JS, XML and Ajax, using HTML, CSS and JS for their purposes in this part were straight forward. Since the requirements suggested to use XML and Ajax, my assumption was that the quiz material should be stored in XML and retrieved using Ajax so that the "sensitive" material (correct answers) could not be easily found in HTML. This part of the assignment was definetly one of the more time-consuming parts in terms of typing/filtering the material for each tutorial.
  3. Part 3: Requirements were not too complicated which was to create a slideshow using the canvas element. I added functionality for a manual/automatic progression of the images and disable the buttons during transitions so that there werem't undesired changes during the transition effects. I had some issues with the setInterval method causing infinite loops because intervals were being created faster than my images could be loaded in. To solve this issue I used the setTimeout method which is the same as setInterval except runs once and I set the timeout at the end of each change on the transition as long as the exit conditions weren't true. Another issue I encountered during the development of this part was that on some browsers/computers, the images would "flicker" or start the transition before loading the next image. My fixes for this were to
    • Use a second Image object that would load the next image and would only run the transition once the next image was loaded
    • Resize my images to be the same length and width so that I would not have to resize each image
    • By resizing it reduced the size of the image that needed to be loaded (MB to kB)

    Finally for the utility tool of my choosing, I decided to make an accumulated depreciation calculator for property, plant and equipment. This calculation was part of an accounting course I took and is an important calculation for accountants. A basic explanation of what accumulated depreciation (AD) is when a company has a vehicle for example, its value decreases as it is in use and AD is to determine how much that vehicle depreciates in value per year. The 2 methods to calculate it is the Straight-Line method and Double-decling method. Here are the following formulas I used: SL Depreciation Expense = (Cost of PPE - Estimated Residual Value)/Estimated Useful Life (Years) &
    DD Dep Expense = Current Starting Carrying Amount x (2 / Estimated Useful Life (Years)) &
    *Ending Carrying Amount = Starting Carrying Amount - Depreciation Expense. One of the rules for these calclations is that the Carrying Amount can not be less than the Estimated Residual Value (value that you would sell the vehicle for at the end of its useful life). For the Straight-line method, it will always end up having an ending carrying amount equal to the residual value. For the Double-declining method, there is a potential that the depreciation expense when subtracted would cause this rule to fail. For example, if we had a Dep Exp = 2000, Starting CA = 4000, and Residual = 3000 then the Ending CA would fail this rule. The proper calculation is to use Depreciation Expense = Starting CA - Residual when * formula is not valid (i.e. the rule above is false). For some documentation on this process see Straight-line or Double-declining (Note: The double declining method online uses a slightly different formula than what I learned from the accounting course but should explain the steps)

Steps to run the code

  1. Download and Setup a local host using XAMPP (see documentation here). For me, the default port for the localhost was already in use so I changed the default port for Apache to 8088 (can be any open port).
  2. Unzip and place the TMA1 folder under the htdocs folder (this path assuming you followed the default installation for XAMPP "C:\xampp\htdocs")
  3. Then type in "localhost:8088/TMA1/tma1.html" assuming you are using port 8088 otherwise change it to the port you are using or you can just put localhost/TMA1/tma1.html if you do not change the default port in step 1.
  4. You can then navigate to each part using the links above or using the links in the header menu. To navigate back to the home page (this page), click the globe in the upper-left corner
